Advancement Import

Files that are imported must have a .csv extension as such make sure that all your values are separated by commas.
It is important to note that when you specify a advancement name that already exists in BSA Database this will update the existing data. To keep existing requirements you must specify them in the import.
When adding requirements please list all requirements for an advancement after the advancement. To signify a new advancement leave an empty space between the last requirement and the name of the next advancement.
The following table will explain what data is valid and how to build our import file. The type and max values are relate to the type of information that will be stored in the associated field (such as: Text or Number) and the maximum number of character allow in each field respectfully.

Advancement Name (Required)
(type, max)		 The name of the advancement.
(Text, 254)
Time Requirement (Optional)

(type, max)		 The time required to be at this rank before advancing to next rank.
(Whole numbers only)
(Number, N/A)
Advancement Image Path (Optional)
(type, max)		 The url of the advancement image. The path should start at the c:/ directory.
(Text, 254)
Service Hours (Optional)
(type, max)		 The number of service hours required for this rank.
(Number, N/A)
Requirement Name (Optional)
(type, max)		 The name of the requirement.
(Text, 44)
Requirement Description (Optional if requirement name is not present)
(type, max)		 The requirement description.
(Text, N/A)

Example

"Advancement Name","Time Requirement","Advancement Image Path","Service Hours"
"Requirement Name","Requirement Description"
